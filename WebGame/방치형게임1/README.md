# 방치형게임1

간단한 웹 기반 방치형 게임 프로토타입입니다.  
`Cookie Clicker` 스타일의 기본 성장 루프를 작은 규모로 구현했고, 클릭 수집부터 자동 생산, 업그레이드, 프레스티지, 저장 기능까지 포함합니다.

## 게임 개요

플레이어는 `별가루`를 모아 생산 건물을 구매하고, 업그레이드를 해금하며 생산 속도를 높입니다.  
누적 별가루가 충분히 쌓이면 `코어 재점화`를 통해 회차를 초기화하고 영구 배율을 얻을 수 있습니다.

핵심 루프:

`클릭 수집 -> 건물 구매 -> 자동 생산 증가 -> 업그레이드 해금 -> 프레스티지 -> 더 빠른 회차`

## 현재 구현 기능

- 클릭으로 별가루를 수집하는 수동 생산 시스템
- 건물 4종을 통한 자동 생산 시스템
- 조건 해금형 업그레이드 시스템
- 프레스티지 기반 `코어 재점화` 시스템
- `localStorage` 기반 자동 저장 / 수동 저장
- 오프라인 접속 보상
- 현재 진행 단계에 맞춘 목표 문구 표시
- 최근 이벤트 5개를 보여주는 로그 시스템

## 실행 방법

브라우저에서 `index.html`을 열면 실행할 수 있습니다.

ES module 구조로 분리되어 있으므로, 브라우저 환경에 따라 로컬 서버 실행이 더 안정적일 수 있습니다.

예시:

```bash
python -m http.server
```

그 뒤 브라우저에서 프로젝트 폴더를 열어 실행합니다.

## 모듈 설명

### `script.js`

앱 진입점입니다. `createGame()`을 호출해 게임을 시작합니다.

### `src/config.js`

게임의 설정값과 정적 데이터가 들어 있습니다.

- 저장 키
- 틱 주기
- 자동 저장 주기
- 오프라인 보상 설정
- 건물 정의
- 업그레이드 정의

### `src/state.js`

기본 게임 상태를 생성합니다.

- 자원 수치
- 누적 수익
- 프레스티지 자원
- 건물 보유량
- 업그레이드 구매 기록
- 로그 초기값

### `src/calculations.js`

게임의 핵심 계산 로직을 담당합니다.

- 클릭당 획득량
- 건물 가격
- 건물 생산량
- 총 자동 생산량
- 프레스티지 획득량
- 오프라인 보상량
- 현재 목표 문구 계산

### `src/formatters.js`

화면에 표시할 숫자와 시간을 포맷합니다.

- 큰 숫자 단위 축약
- 마지막 저장 시간 표시

### `src/storage.js`

저장 및 불러오기 로직을 담당합니다.

- `localStorage` 저장
- 저장 데이터 복원
- 오프라인 보상 적용
- 저장 초기화

### `src/dom.js`

필요한 DOM 요소를 한곳에서 수집합니다.

### `src/render.js`

화면 렌더링 전용 모듈입니다.

- 자원 영역 렌더링
- 건물 카드 렌더링
- 업그레이드 카드 렌더링
- 상태 로그 렌더링
- 버튼 활성화 상태 반영

### `src/game.js`

게임 실행 흐름을 담당하는 메인 모듈입니다.

- 클릭 이벤트 연결
- 건물 구매 처리
- 업그레이드 구매 처리
- 프레스티지 처리
- 전체 초기화 처리
- 자동 생산 틱 처리
- 자동 저장 실행
